# Code of Conduct

## Our Standard

AetherOS should be a focused, ambitious, and respectful project space. Contributors are expected to help keep discussion useful, specific, and welcoming.

Examples of encouraged behavior:

- Be clear and constructive when reporting bugs.
- Explain tradeoffs when proposing architecture changes.
- Respect project constraints such as vanilla TypeScript, Tauri, Rust, and no React/Next/Tailwind.
- Keep security discussions responsible and transparent.
- Assume good intent while still being precise about problems.

Examples of unacceptable behavior:

- Harassment, insults, threats, or discriminatory language.
- Publishing private security details before maintainers can respond.
- Submitting malware-like behavior, stealth bypass features, or unsafe destructive code.
- Repeatedly derailing technical discussions.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or contributions that violate this code of conduct. Repeated or severe violations may result in blocked participation.

## Reporting

If the project is hosted on GitHub, report conduct concerns to the repository maintainer. For security issues, use the process in `SECURITY.md` instead.
